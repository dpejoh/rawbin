7 key improvements

1
Command palette + keyboard-first navigation
Add ⌘K to instantly search across all clipboards, files, and keybox entries. This is the single biggest UX win — it's what makes Raycast and Linear feel "fast" even with large datasets.
⌘K shortcut fuzzy search MDUI Dialog
2
3-pane layout: nav → list → detail
Replace the current single-panel view with a master-detail layout. Clicking a clipboard item shows its content side-by-side without a full page switch. Zero navigation cost = faster workflows.
no full reload context preserved
3
Syntax detection + colored content badges
Auto-detect content type (XML, JSON, PEM, text, base64) from the first 200 bytes. Show a colored badge on each list item and apply basic token coloring in the preview area. No external library needed — a few regex passes cover 90% of cases.
type badges token highlight regex-based, zero deps
4
One-click URL row with copy + QR actions
Replace the raw URL textarea with a compact pill that has a live URL, a Copy button, and a QR popover. Keep the UUID alias URL visible on hover. Developers share URLs constantly — make it instant.
1-click copy QR popover MDUI Tooltip
5
Sidebar identity: logo + status + settings
Add the yuribin wordmark + icon, a connected YubiKey status dot, and a settings shortcut at the bottom. Grouped nav labels ("Storage", "Settings") add visual rhythm. The sidebar currently has no personality — this anchors the whole app.
brand identity YubiKey status MDUI Navigation rail
6
Files section: icon grid view + architecture chips
The sqlite3 folder view is functional but flat. Add an icon-per-type grid, show file size + date as secondary meta, and display architecture tags (arm64, x86_64…) as colored chips. Add a drag-to-upload dropzone for each arch folder.
grid + list toggle arch chips drag-and-drop upload
7
Keybox: obfuscated preview + reveal on hold
The current Keybox shows the full PEM key in plain text in the UI — a security UX anti-pattern. Mask it by default (show only first/last 8 chars + ● dots). Reveal only on "hold to show" button press. Add a copy-without-revealing action. This is how 1Password, Bitwarden, and Infisical handle secrets.
secret masking hold-to-reveal copy-without-reveal
